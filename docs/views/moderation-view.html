Moderation Logs

Keep your moderators informed with automatic event logging — member joins and leaves, deleted messages, and edited messages, all delivered to your chosen channels as clean timestamped embeds.

Overview

Acosmibot's moderation logging system captures key server events and posts detailed embeds to your designated log channels. Every log entry includes timestamps, user information, and relevant context so your mod team always knows what happened, when, and to whom.

Key Features

  • Automatic logging for member joins, leaves, kicks, and bans
  • Message delete logs with original content and channel context
  • Message edit logs showing before and after content side-by-side
  • Route each event type to its own dedicated channel
  • Toggle individual event types on or off independently
  • All logs delivered as timestamped Discord embeds with user info

Logged Events

Member Join
A log embed is posted whenever a new member joins the server. The entry includes the member's username, account creation date, and the exact time they joined — useful for spotting newly created accounts.

Log includes:

Member mention • Account age • Join timestamp
Member Leave
A log is posted when a member leaves, is kicked, or is banned. Where possible, the bot distinguishes between a voluntary departure and an administrative action, including the responsible moderator and any reason provided.

Log includes:

Member mention • Action type (leave / kick / ban) • Moderator • Reason
Message Delete
When a non-bot message is deleted, a log embed is posted showing the original message content, the author, and the channel it was deleted from. Bot messages are excluded to keep logs clean.

Log includes:

Author • Channel • Original message content • Delete timestamp
Message Edit
When a non-bot message is edited, a log embed is posted showing the content before and after the edit. Both versions are displayed together so moderators can quickly review changes.

Log includes:

Author • Channel • Before content • After content • Edit timestamp

How to Set It Up

1

Create Your Log Channels

Create one shared moderation channel, or separate channels for each event type (e.g. #mod-joins, #mod-messages). The bot needs "Send Messages" and "Embed Links" permissions in every log channel.

2

Assign Channels in the Dashboard

Open your /server dashboard and navigate to the Moderation section. Assign each event type to the channel where you want its logs to appear. You can point multiple event types at the same channel if you prefer a unified feed.

3

Enable the Events You Want

Toggle each event type independently. If your server is high-traffic and message-edit logs would be noisy, you can disable that type while keeping others active. Changes take effect immediately.

4

Verify and Test

Have a test account join and send a message, then edit and delete it. Confirm that logs appear in the correct channels with the right information before announcing the setup to your mod team.

Tips for Effective Logging

Best Practices

  • Use separate channels for each event type to keep logs easy to scan
  • Restrict log channels to moderators only — they often contain sensitive message content
  • Pin a brief channel description so moderators know what each log channel covers
  • Consider a single unified channel for smaller servers where log volume is low
  • Pair moderation logs with Discord's built-in audit log for the most complete picture

Things to Keep in Mind

  • Messages deleted before the bot was added to the server cannot be recovered
  • Discord does not expose who deleted a message unless it was deleted by a moderator — bulk deletes may show as unknown
  • Very long messages are truncated in logs to fit Discord's embed character limits
  • The bot must be online when an event occurs to log it — events during downtime are missed

Troubleshooting

Q: Logs aren't appearing in my channel

A: First confirm a channel is assigned for that event type in the dashboard and that the toggle is enabled. Then check that the bot has "Send Messages" and "Embed Links" permissions in the target channel.

Q: Member leave logs aren't distinguishing kicks from voluntary leaves

A: The bot checks Discord's audit log to identify kicks and bans. Make sure the bot role has "View Audit Log" permission. Without it, all departures will appear as voluntary leaves.

Q: Message delete logs are showing "Content unavailable"

A: Discord does not cache every message. If a message was sent before the bot started watching the channel, or if Discord's cache expired, the original content cannot be retrieved.

Q: Can I log events to a thread instead of a channel?

A: At this time, log channels must be standard text channels. Thread support may be added in a future update.

Q: Will the bot log its own messages if they're deleted?

A: No. Bot messages are excluded from message delete and edit logs to prevent unnecessary noise.
