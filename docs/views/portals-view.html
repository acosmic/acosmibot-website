Cross-Server Portals

Open a temporary communication bridge between two Discord servers and let communities talk to each other in real time.

Overview

Cross-Server Portals let any two servers with Acosmibot open a live two-minute communication session. Messages sent through the portal are relayed between both servers instantly — it's like a direct phone line between communities. Portals are credit-powered, making each connection a meaningful interaction rather than background noise.

Key Features

  • Two-minute real-time messaging sessions between servers
  • Messages flow both directions — both communities can participate
  • Searchable portal directory for discovering other servers
  • Credit cost per connection (configurable per server)
  • Admins can control portal availability, channel, and visibility

How It Works

1

Find a server

Use /portal-search to browse the public portal directory. Any server that has opted into public listing will appear here. Search by name or topic to find communities you want to connect with.

2

Open a portal

Run /portal-open <server id> to initiate the connection. This spends credits from your balance (default 1,000, but each server can set their own cost). The portal opens immediately if the target server has portals enabled.

3

Communicate

Use /portal-send <message> to send messages through the active portal. Both servers see every message relayed in real time in their designated portal channel. The session lasts two minutes.

4

Portal closes

After two minutes the portal automatically closes. You can also close it early with /portal-close. Credits are spent on open — there's no refund for closing early.

Commands

/portal-search <query>
Search the directory of servers that have made their portal publicly visible. Returns a list of matching servers with their display names and portal credit costs.

Examples:

/portal-search gaming /portal-search anime
/portal-open <server id>
Open a portal to another server using its Discord server ID. Spends credits from your balance. The default cost is 1,000 credits, though each server can configure their own rate.

Example:

/portal-open 123456789012345678
/portal-send <message>
Send a message through the currently active portal. The message is relayed to the other server's portal channel in real time.

Example:

/portal-send Hey from our server! Anyone want to play tonight?
/portal-status
Check whether a portal is currently open in this server and see how much time remains in the session.

Example:

/portal-status
/portal-close
Close the active portal session early. Both servers are notified that the portal has closed. Credits are not refunded for early closure.

Example:

/portal-close

Server Configuration (Admin)

Server administrators can control exactly how portals work in their community. All portal settings are managed through the /server dashboard.

Setting Description
Enable / Disable Portals Turn portal functionality on or off for your server entirely
Portal Channel Designate which channel receives and displays incoming portal messages
Public Listing Opt in to the portal directory so other servers can find and connect to you
Display Name Set a custom name shown in search results (defaults to your server name)
Credit Cost Configure how many credits it costs for another server to open a portal to yours

Getting Listed in the Directory

  • Enable portals in your server dashboard
  • Designate a portal channel where messages will appear
  • Toggle "Public Listing" to appear in /portal-search results
  • Set a friendly display name so other communities recognize you
  • Optionally adjust the credit cost to open a portal to your server

Credits & Costs

Opening a portal costs the initiating user credits. The default cost is 1,000 credits, but each server can set their own price for incoming portals. Check a server's cost with /portal-search before connecting.

Credit Spending Notes

  • Credits are deducted from the user who runs /portal-open
  • Credits are spent at the moment the portal opens — not refunded if closed early
  • Each server sets their own incoming portal cost independently
  • Only one portal can be active per server at a time

Frequently Asked Questions

Q: Can both servers see who sent each message?

A: Yes. Relayed messages include the sender's display name and server so both communities know who they're talking to.

Q: What happens if a server has portals disabled?

A: You won't be able to open a portal to that server, and it won't appear in the public directory. The command will return an error if you try to connect using its server ID directly.

Q: Can admins moderate portal messages?

A: Admins can close the portal at any time with /portal-close and can disable portals entirely from the dashboard if a connection becomes problematic.

Q: Is there a cooldown between portals?

A: Only one portal can be active per server at a time. Once a session ends, you can open a new one immediately.
