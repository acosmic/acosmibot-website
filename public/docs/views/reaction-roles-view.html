Reaction Roles

Let members self-assign roles by reacting to a message, clicking a button, or choosing from a dropdown — no admin intervention required.

Overview

Reaction Roles let your members take control of their own role assignments. Instead of admins manually handing out roles, members can pick the ones that apply to them — whether that's game preferences, notification opt-ins, color roles, or anything else you can imagine. Set it up once and it just works.

Key Features

  • Three interaction types: emoji reactions, buttons, and dropdowns
  • One emoji or button can assign multiple roles simultaneously
  • Toggle role removal on or off per configuration
  • Works with any roles the bot has permission to manage
  • Fully configured through the dashboard — no commands needed
  • Changes apply instantly without restarting the bot

Interaction Types

🎭 Emoji Reactions

Members react to a designated message with a specific emoji to receive the mapped role. Remove the reaction to remove the role (if role removal is enabled in your configuration). This is the classic reaction roles experience.

🔘 Buttons

The bot posts a message with clickable buttons. Members click a button to toggle their role on or off. Buttons are more visually prominent than reactions and work well for servers with members who might miss small emoji options.

📋 Dropdowns

A dropdown (select menu) is attached to a message. Members open the menu and pick from a list of available roles. Great for servers with many role options — keeps the interface clean and organized.

Setup

1

Open the Dashboard

Go to your server dashboard and navigate to the Reaction Roles section. This is where all your reaction role configurations live.

2

Choose an Interaction Type

Select whether you want to use Emoji Reactions, Buttons, or a Dropdown for this configuration. Each type suits different use cases.

3

Configure the Mapping

Specify the target message, then map each emoji, button, or dropdown option to one or more roles. A single interaction can assign multiple roles at once.

4

Set Role Removal Behavior

Decide whether removing the reaction (or clicking again) should also remove the role. You can toggle this on or off per configuration depending on your use case.

5

Save and Go Live

Save your configuration from the dashboard. The reaction role goes live immediately — no bot restart needed.

Configuration Options

Setting Description
Interaction Type Emoji reaction, button, or dropdown
Target Message The message members interact with to receive roles
Emoji / Label The emoji to react with, or the button/option label
Role(s) One or more roles assigned by this interaction
Role Removal Whether un-reacting or re-clicking removes the role

Best Practices

💡 Tips for a Great Setup

  • Pin your reaction role message so members can always find it
  • Use descriptive button labels or emoji that clearly indicate the role
  • Group related roles together in one message for a cleaner experience
  • Use dropdowns when you have more than 5–6 role options to keep things tidy
  • Test your setup in a private channel before publishing to the server
  • Make sure the bot's role is higher than the roles it needs to assign

⚠️ Bot Permission Requirements

The bot can only assign roles that are lower in the role hierarchy than its own highest role. If the bot can't assign a role, check that the bot's role sits above the target role in Server Settings → Roles.

Troubleshooting

Q: Members react but don't receive the role

A: Check that the bot has the Manage Roles permission and that its role is positioned above the target role in the server's role hierarchy.

Q: Removing a reaction doesn't remove the role

A: Role removal must be enabled in your reaction role configuration on the dashboard. Check that setting and save the configuration again.

Q: The bot's reactions disappeared from the message

A: This can happen if the bot lost the Add Reactions or Read Message History permission in that channel. Restore those permissions and the bot will re-add its reactions.

Q: Can one emoji assign more than one role?

A: Yes! A single emoji, button, or dropdown option can be mapped to multiple roles at once. Configure this from the dashboard when setting up the mapping.

Q: Can I use custom server emojis?

A: Yes. The bot supports both standard Unicode emojis and custom server emojis for reaction-type configurations.
