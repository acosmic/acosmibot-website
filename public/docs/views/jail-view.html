Jail System

Temporarily restrict members to a designated jail channel — a lightweight moderation tool for cooling down situations without a full ban or kick.

⚠️ Feature Currently Unavailable

The Jail system is currently disabled while it undergoes improvements. The commands documented below are not active at this time. This page reflects the planned behavior for when the feature returns. Check back or join the support server for updates.

Overview

The Jail system gives moderators a middle-ground option between a warning and a kick. When a member is jailed, they lose access to all regular server channels and are confined to a designated #jail channel for the duration of the sentence. Once the time expires — or a moderator releases them early — full access is automatically restored.

Key Features

  • Temporarily restrict members to a single #jail channel
  • Set a custom duration per sentence — seconds to days
  • Automatic release when the timer expires
  • Moderators can release members early with /unjail
  • The jailed member can still read and send messages in the jail channel
  • Full channel access is restored automatically on release

Commands

/jail <user> <duration> [reason]
Restrict the target member to the jail channel for the specified duration. The member immediately loses access to all other channels. An optional reason can be provided and will be logged. Requires Moderator or Administrator permissions.

Examples:

/jail @username 10m /jail @username 1h Spamming in #general /jail @username 1d Repeated rule violations
/unjail <user>
Release a jailed member before their sentence expires. Their channel access is restored immediately. Requires Moderator or Administrator permissions.

Example:

/unjail @username
/jaillist
View all currently jailed members in the server, including how much time remains on each sentence. Requires Moderator or Administrator permissions.

Example:

/jaillist

Setup

Before using the Jail system, a server admin needs to designate the jail channel and configure the jail role in the server dashboard.

1

Create a #jail channel

Create a dedicated channel (e.g. #jail) where jailed members will be confined. This channel should be visible only to jailed members, moderators, and the bot.

2

Configure in the dashboard

Open the /server dashboard, navigate to the Moderation section, and select your jail channel. Acosmibot will automatically manage the jail role and channel permissions from here.

3

Verify bot permissions

Ensure Acosmibot has Manage Roles and Manage Channels permissions. These are required to assign the jail role and override channel access for jailed members.

4

Start jailing

Moderators can now use /jail @member <duration>. The bot handles all role assignment and permission overrides automatically.

How It Works Technically

When a member is jailed, Acosmibot assigns them a Jailed role. This role uses Discord channel permission overrides to deny access to all regular channels while allowing read and write access to the designated jail channel. On release — automatic or manual — the role is removed and all access is restored to normal.

Permission Flow

  • Bot assigns the Jailed role to the member
  • The Jailed role denies View Channel on all regular channels via role permissions
  • The jail channel explicitly allows View Channel and Send Messages for the Jailed role
  • When released, the Jailed role is removed and access returns to normal

Role Hierarchy Requirement

  • Acosmibot's role must be positioned above the Jailed role in the server role list
  • Acosmibot cannot jail members whose highest role is ranked above the bot's role
  • Server owners cannot be jailed
  • Admins with "Administrator" permission may bypass the jail channel restriction — consider this when structuring roles

Best Practices

Tips for Moderators

  • Always include a reason so both the member and your mod team understand the context
  • Use short durations (10–60 minutes) for first offenses or minor disruptions
  • Reserve longer sentences for repeat offenders or more serious violations
  • Check /jaillist regularly so no one is accidentally left jailed past their intended sentence
  • Consider posting a pinned message in #jail explaining the process and what the member needs to do to be released

Things to Avoid

  • Don't jail members as a joke — it removes real access and can be disorienting
  • Avoid indefinite jails; if a member needs a permanent restriction, a ban or role removal is more appropriate
  • Don't set excessively long durations without a clear policy — consistency builds trust in your moderation team

Frequently Asked Questions

Q: Can a jailed member see that they're in jail?

A: Yes. They can read and send messages in the jail channel, so they're aware of their situation. The bot posts a message in the jail channel notifying them of the duration and reason.

Q: What if the bot goes offline during a jail sentence?

A: The jail timer is stored in the database. When the bot comes back online, it checks for expired sentences and releases any members whose time has passed.

Q: Can jailed members still use DMs?

A: Yes. The jail system only affects server channel access. Direct messages are unaffected.

Q: Can I jail someone who already has a role higher than mine?

A: No. Discord's role hierarchy prevents the bot from assigning roles to members who outrank the bot in the role list. Reorder the bot's role above the target's highest role in Server Settings → Roles to resolve this.

Q: Is there a mod log for jail actions?

A: Yes, if you have a mod log channel configured in the dashboard, all jail and unjail actions are logged there with the moderator's name, target, duration, and reason.
