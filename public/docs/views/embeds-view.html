Better Embeds

Build rich, beautifully formatted embed messages for announcements, information cards, and custom bot responses.

Overview

Acosmibot's embed builder lets you craft Discord embeds with full control over every visual element — from the color accent to images, footers, and labeled field sections. Embeds are validated before saving so you'll never end up with broken output in your channels.

What You Can Customize

  • Title — The bold heading displayed at the top of the embed
  • Description — The main body text supporting Discord markdown
  • Color — A hex color code that sets the left border accent
  • Fields — Any number of labeled sections, with optional inline layout
  • Thumbnail — A small image displayed in the top-right corner
  • Image — A full-width image displayed at the bottom of the embed
  • Author — Author display name and optional icon
  • Footer — Small descriptive text shown at the very bottom

Embed Structure

Embeds are defined using a JSON structure. Below is a reference showing all available fields. Only include the fields you need — every field is optional.

{ "title": "Welcome to the Server!", "description": "We're glad you're here. Read the rules and enjoy your stay.", "color": "#5865F2", "author": { "name": "Server Staff", "icon_url": "https://example.com/icon.png" }, "thumbnail": { "url": "https://example.com/thumbnail.png" }, "fields": [ { "name": "Rules", "value": "See #rules for guidelines.", "inline": true }, { "name": "Support", "value": "Ask in #help.", "inline": true } ], "image": { "url": "https://example.com/banner.png" }, "footer": { "text": "Last updated April 2026" } }

Field Reference

Field Type Description Limit
title string Bold heading at the top of the embed 256 chars
description string Main body text; supports Discord markdown 4096 chars
color hex string Left border accent color (e.g. #FF5733) —
author.name string Author name shown above the title 256 chars
author.icon_url URL Small icon shown next to the author name —
thumbnail.url URL Small image in the top-right corner —
fields array Labeled sections with name, value, and optional inline flag 25 fields
image.url URL Full-width image at the bottom of the embed —
footer.text string Small text at the very bottom 2048 chars

Inline Fields

When a field has "inline": true, Discord will attempt to display it side-by-side with adjacent inline fields — typically two or three per row depending on screen width. This is useful for compact stat displays or comparison tables.

Inline Layout Tips

  • Discord displays up to 3 inline fields per row on desktop
  • A non-inline field always starts a new row
  • Keep inline field values short to avoid layout issues on mobile
  • You can mix inline and non-inline fields freely

Validation & Saving

Before an embed is saved, Acosmibot validates the JSON structure and field limits. If something is malformed — an invalid hex color, a field that's too long, or a missing required value — you'll receive a clear error message explaining what needs to be fixed.

Common Validation Errors

  • Color must be a valid hex code starting with # (e.g. #5865F2)
  • Image and thumbnail values must be valid HTTPS URLs
  • Field names and values cannot be empty strings
  • Total embed character count cannot exceed 6000
  • Maximum of 25 fields per embed

Where Embeds Are Used

Embeds can be attached to custom commands so the bot responds with a rich card instead of plain text. They're also used in bot-generated announcements, welcome messages, and automated notifications throughout your server.

Common Use Cases

  • Welcome messages with server rules and links
  • Announcement cards for events or updates
  • Info commands that display structured data neatly
  • Role descriptions or FAQ cards pinned in channels
  • Giveaway or contest announcements

Frequently Asked Questions

Q: Can I use Discord markdown in embed descriptions?

A: Yes. Descriptions and field values support standard Discord markdown including bold, italic, code blocks, and hyperlinks.

Q: Can I include multiple images?

A: Each embed supports one thumbnail (top-right) and one large image (bottom). To display more, you would need multiple embeds.

Q: Do embeds work on mobile?

A: Yes, though inline field layout and image sizes may render differently on smaller screens. Test your embeds across devices when presentation matters.

Q: Where do I build and save embeds?

A: Embeds are configured through the /server dashboard embed builder, which includes a live preview so you can see exactly how your embed will look before saving.
