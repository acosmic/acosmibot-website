Custom Commands

Create your own commands with text or rich embed responses — give your server a personal touch without writing any code.

Overview

Custom Commands let you define your own bot responses triggered by a prefix and a name you choose. Whether it's a quick FAQ answer, a server rules summary, a fun catchphrase, or a beautifully formatted embed with your server's branding — if members type the command, the bot delivers the response instantly.

Key Features

  • Trigger commands with a prefix (typically !) and a name you define
  • Two response types: plain text or rich Discord embeds
  • Embeds support title, description, color, fields, thumbnail, image, author, and footer
  • Changes take effect immediately — no bot restart needed
  • Free tier: 1 custom command per server
  • Premium tier: up to 25 custom commands per server

How It Works

1

Open the Dashboard

Navigate to your server dashboard and open the Custom Commands section. This is where you create and manage all your commands.

2

Create a New Command

Give your command a name — this is what members type after the prefix. For example, a command named rules is triggered by typing !rules in chat.

!rules
3

Choose a Response Type

Select Text for a simple message, or Embed for a rich, formatted response. Embeds let you add a title, color, fields, images, and more.

4

Write Your Response

For text responses, type your message. For embeds, fill in whichever fields you need — all embed fields are optional except the description. The preview in the dashboard shows exactly how it will look.

5

Save and Use It

Save the command from the dashboard. It's live instantly — members can use it in any channel the bot has access to right away.

Response Types

💬 Text Response

A plain text message sent by the bot when the command is triggered. Fast and simple — perfect for short answers, links, or quick information like a server invite or social media handle.

Our Discord rules can be found in #rules. Please read them before posting!

🎨 Embed Response

A rich Discord embed with full formatting support. Use embeds when you want your response to stand out or convey structured information clearly.

  • Title — Bold heading at the top of the embed
  • Description — Main body text (supports Discord markdown)
  • Color — Left-border accent color of the embed
  • Fields — Labeled sections, optionally displayed inline
  • Thumbnail — Small image in the top-right corner
  • Image — Large image below the description
  • Author — Name and icon shown above the title
  • Footer — Small text and icon at the bottom

Configuration

Custom commands are managed entirely through the server dashboard. There are no slash commands for creating or editing them.

Setting Description
Command Name The word members type after the prefix to trigger the command
Prefix The character that precedes the command name (default: !)
Response Type Text or Embed
Response Content The message or embed data returned when the command is used

Best Practices

💡 Tips for Great Custom Commands

  • Keep command names short and intuitive — members shouldn't have to guess
  • Pin a message listing your server's custom commands so members know they exist
  • Use embeds for longer responses — they're easier to read than walls of text
  • Use the color field in embeds to match your server's branding
  • Avoid special characters or spaces in command names — stick to letters and hyphens
  • Update commands from the dashboard whenever information changes; it's instant

⚠️ Command Name Conflicts

Custom command names cannot conflict with Acosmibot's built-in slash commands. If a name is already in use, choose a different one or make it unique (e.g., !server-rules instead of !rules).

Troubleshooting

Q: I typed the command but the bot didn't respond

A: Make sure you're using the correct prefix (default !) and the exact command name as configured. Also confirm the bot has permission to read messages and send messages in that channel.

Q: I can't create more commands — it says I've hit the limit

A: Free servers are limited to 1 custom command. Upgrade to Premium to create up to 25. If you're already on Premium and at 25, consider combining related commands into a single embed with multiple fields.

Q: My embed looks different in the dashboard preview versus Discord

A: Minor rendering differences can occur between the dashboard preview and Discord's client. The structure and content will always match — only subtle visual details may vary slightly.

Q: Can I restrict a custom command to certain channels or roles?

A: Custom commands currently respond in any channel the bot can read and write in. To limit usage, restrict the bot's channel permissions to only the channels where you want commands available.

Q: Can members trigger custom commands in DMs?

A: No. Custom commands only work in server channels where the bot is present.
